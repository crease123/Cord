# Electron 主进程与渲染进程详解

## 一、核心概念

Electron 应用由两种类型的进程组成：**主进程（Main Process）** 和 **渲染进程（Renderer Process）**。

```
┌─────────────────────────────────────────────────────────────────┐
│                      Electron 应用                               │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  ┌─────────────────────┐    ┌─────────────────────────────────┐│
│  │   主进程 (Main)      │    │     渲染进程 (Renderer)          ││
│  │   Node.js 环境       │◄──►│     浏览器环境 (Chromium)        ││
│  │                      │    │                                  ││
│  │  • 系统级操作         │    │  • 用户界面 (UI)                 ││
│  │  • 文件读写           │    │  • DOM 操作                      ││
│  │  • 窗口管理           │    │  • Vue/React 组件                ││
│  │  • 托盘/菜单          │    │  • 用户交互                      ││
│  │  • 网络服务           │    │                                  ││
│  └─────────────────────┘    └─────────────────────────────────┘│
│                                                                  │
│              通信方式: HTTP API / IPC / JSON-RPC                  │
└─────────────────────────────────────────────────────────────────┘
```

### 1.1 主进程 (Main Process)

- **数量**：整个应用只有 **1 个**主进程
- **运行环境**：Node.js 环境，可以使用所有 Node.js API
- **职责**：
  - 创建和管理应用窗口 (`BrowserWindow`)
  - 控制应用生命周期 (`app` 模块)
  - 访问操作系统功能（文件系统、剪贴板等）
  - 创建系统托盘、菜单
  - 启动本地服务器

### 1.2 渲染进程 (Renderer Process)

- **数量**：每个窗口/页面有 **1 个**渲染进程
- **运行环境**：Chromium 浏览器环境
- **职责**：
  - 渲染网页内容 (HTML/CSS/JS)
  - 运行前端框架 (Vue 3)
  - 处理用户交互
  - 调用 Web API

## 三、主进程代码详解

### 3.1 主进程入口 - `src/main/app.ts`

主进程负责创建窗口并加载渲染进程的页面：

```typescript
// 导入 Electron 模块（只能在主进程中使用）
import { protocol, app, Menu, Tray, BrowserWindow } from 'electron'

// 创建窗口函数
const createWindow = () => {
  win = new BrowserWindow({
    maximizable: true,
    show: false,
    minWidth: 940,
    minHeight: 500,
    frame: false,                    // 无边框窗口
    backgroundColor: '#282a2b',
    webPreferences: {
      webSecurity: false,
      nodeIntegration: true,         // 允许渲染进程使用 Node.js
      contextIsolation: false,       // 关闭上下文隔离
    },
  })

  // 加载渲染进程的页面
  win.loadURL(getUrl())  // 例如: yank-note://localhost:3044
}

// 应用生命周期管理
app.on('ready', () => {
  startup()           // 启动前置任务
  serve()             // 启动 HTTP 服务器
  showWindow()        // 显示窗口
  showTray()          // 显示系统托盘
})
```

**关键点**：
- `app` 模块控制应用生命周期
- `BrowserWindow` 创建应用窗口
- `webPreferences.nodeIntegration: true` 允许渲染进程使用 Node.js API

### 3.2 HTTP 服务器 - `src/main/server/index.ts`

主进程运行一个 Koa HTTP 服务器，为渲染进程提供 API：

```typescript
import Koa from 'koa'
import * as file from './file'

const server = (port = 3000) => {
  const app = new Koa()

  // 文件操作 API
  app.use(async (ctx, next) => {
    if (ctx.path === '/api/file' && ctx.method === 'GET') {
      const { repo, path } = ctx.query
      const content = await file.read(repo, path)  // 读取文件
      ctx.body = { status: 'ok', data: { content } }
    }
  })

  // 启动服务器监听
  server.listen(port, '127.0.0.1')
}
```

**为什么需要 HTTP 服务器？**
1. 文件操作必须在 Node.js 环境（主进程）执行
2. 渲染进程通过 HTTP 请求调用这些 API
3. 这种设计也支持浏览器直接访问（非 Electron 环境）

## 四、渲染进程代码详解

### 4.1 渲染进程入口 - `src/renderer/index.ts`

渲染进程使用 Vue 3 创建用户界面：

```typescript
import { createApp } from 'vue'
import Main from '@fe/Main.vue'

// 创建 Vue 应用
const app = createApp(Main)

// 挂载到 DOM
app.mount('#app')
```

### 4.2 环境检测 - `src/renderer/support/env.ts`

渲染进程需要判断当前运行环境：

```typescript
// 获取 Node.js 的 process 对象（如果在 Electron 中可用）
export const nodeProcess: NodeJS.Process = window.process

// 判断是否在 Electron 环境中
export const isElectron = !!(nodeProcess?.versions?.electron)

// 判断操作系统
export const isMacOS = /macintosh|mac os x/i.test(navigator.userAgent)
export const isWindows = /win64|win32/i.test(navigator.userAgent)

// 获取 Electron Remote 模块（访问主进程功能）
export function getElectronRemote () {
  if (!isElectron) {
    throw new Error('not in electron')
  }
  return nodeRequire('@electron/remote')
}
```

### 4.3 API 调用 - `src/renderer/support/api.ts`

渲染进程通过 HTTP 请求调用主进程提供的 API：

```typescript
// 基础 HTTP 请求函数
export async function fetchHttp (input: RequestInfo, init?: RequestInit) {
  const response = await fetch(input, {
    ...init,
    headers: {
      'Authorization': 'Bearer ' + JWT_TOKEN,
      ...init?.headers
    }
  })
  return response.json()
}

// 读取文件示例
export async function readFile (repo: string, path: string) {
  const result = await fetchHttp(`/api/file?repo=${repo}&path=${path}`)
  return result.data.content
}

// 在主进程执行代码（RPC 调用）
export async function rpc (code: string) {
  const { data } = await fetchHttp('/api/rpc', {
    method: 'POST',
    body: JSON.stringify({ code })
  })
  return data
}

// 示例用法：
// await rpc("return require('os').platform()")  // 返回 'win32'
```

## 五、进程间通信方式

Yank Note 使用 **三种** 方式实现进程间通信：

### 5.1 HTTP API（主要方式）

```
渲染进程                           主进程
   │                                │
   │  HTTP GET /api/file?path=xxx   │
   │  ─────────────────────────────►│
   │                                │ 读取文件
   │  { status: 'ok', data: {...} } │
   │  ◄─────────────────────────────│
   │                                │
```

**优点**：
- 前后端解耦，可独立开发
- 支持浏览器直接访问

### 5.2 IPC 通信（Electron 原生）

```typescript
// 主进程 - 监听消息
import { ipcMain } from 'electron'
ipcMain.on('jsonrpc', (event, message) => {
  // 处理来自渲染进程的消息
})

// 渲染进程 - 发送消息
import { ipcRenderer } from 'electron'
ipcRenderer.send('jsonrpc', { method: 'xxx', params: [] })
```

### 5.3 JSON-RPC（双向调用）

Yank Note 封装了 JSON-RPC 实现双向通信：

**主进程调用渲染进程** - `src/main/jsonrpc.ts`：
```typescript
// 主进程作为 RPC 客户端
export let jsonRPCClient: JSONRPCClient<{ ctx: Ctx }>

// 调用渲染进程的方法
jsonRPCClient.call.ctx.setting.showSettingPanel()
jsonRPCClient.call.ctx.doc.switchDocByPath(path)
```

**渲染进程响应主进程** - `src/renderer/support/jsonrpc.ts`：
```typescript
// 渲染进程作为 RPC 服务端
class ElectronRendererServerChannel {
  constructor () {
    this.ipcRenderer = nodeRequire('electron').ipcRenderer
  }

  setMessageHandler (callback) {
    // 监听来自主进程的调用
    this.ipcRenderer.on('jsonrpc', (event, message) => {
      callback(message)
    })
  }
}
```

## 七、对比总结表

| 特性 | 主进程 | 渲染进程 |
|------|--------|----------|
| **数量** | 1 个 | 每个窗口 1 个 |
| **运行环境** | Node.js | Chromium 浏览器 |
| **入口文件** | `src/main/app.ts` | `src/renderer/index.ts` |
| **可用 API** | Node.js + Electron | Web API + (可选) Node.js |
| **文件操作** | ✅ 直接访问 | ❌ 需通过 API |
| **UI 渲染** | ❌ | ✅ |
| **示例模块** | `fs`, `path`, `child_process` | Vue, DOM, CSS |

## 九、常见问题

### Q1: 为什么不能直接在渲染进程读写文件？

虽然 Yank Note 配置了 `nodeIntegration: true`，理论上可以直接使用 Node.js API，但：
1. 安全性考虑：将文件操作集中在主进程更安全
2. 架构清晰：API 设计使得代码也能在浏览器中运行
3. 可测试性：HTTP API 更容易进行单元测试

### Q2: 什么时候使用 IPC，什么时候使用 HTTP？

- **HTTP API**：常规数据操作（读写文件、获取设置等）
- **IPC/JSON-RPC**：需要主进程主动通知渲染进程的场景（如打开文件对话框后通知切换文件）

### Q3: `@electron/remote` 是什么？

`@electron/remote` 允许渲染进程调用主进程模块，如：
```typescript
const remote = getElectronRemote()
const win = remote.getCurrentWindow()  // 获取当前窗口对象
win.minimize()  // 最小化窗口
```
