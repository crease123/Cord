## 1. 为什么学习 TypeScript

TypeScript 是 JavaScript 的超集，添加了**静态类型系统**。在 Yank Note 项目中，100% 使用 TypeScript 编写。

**TypeScript 的优势：**
- ✅ **编译时错误检查**：在代码运行前发现错误
- ✅ **智能代码补全**：IDE 可以提供精确的代码提示
- ✅ **代码可读性**：类型本身就是文档
- ✅ **重构安全**：修改代码时编译器会检查所有引用

```typescript
// 字符串
let name: string = 'Yank Note'
```

```typescript

// 方式二：Array<类型>
let ports: Array<number> = [3000, 3001, 3002]
```

### 2.3 对象类型

```typescript
// 直接定义对象结构
let config: { name: string; port: number } = {
  name: 'server',

```

### 2.4 any 和 unknown

```typescript
// any：关闭类型检查（尽量避免使用）
```

## 3. 接口 (Interface)

接口用于定义对象的**形状（Shape）**，是 TypeScript 最常用的类型定义方式。

### 3.1 基本接口

**来自项目 `src/share/types.ts`：**

```typescript
// 定义路径项接口
export interface PathItem {
  repo: string;    // 仓库名
  path: string;    // 文件路径
}

// 定义文件统计信息接口
export interface FileStat {
  mtime: number,      // 修改时间
  birthtime: number,  // 创建时间
  size: number,       // 文件大小
}
```

**使用接口：**

```typescript
// 创建符合 PathItem 接口的对象
const myFile: PathItem = {
  repo: 'main',
  path: '/docs/readme.md'
}

// 错误示例（缺少属性会报错）
const badFile: PathItem = {
  repo: 'main'
  // 错误：缺少 path 属性
}
```

### 3.2 可选属性

使用 `?` 标记可选属性：

```typescript
export interface Doc extends BaseDoc {
  name: string;
  content?: string;        // 可选：内容
  passwordHash?: string;   // 可选：密码哈希
  contentHash?: string;    // 可选：内容哈希
  stat?: FileStat,         // 可选：文件统计
  writeable?: boolean,     // 可选：是否可写
  status?: 'loaded' | 'save-failed' | 'saved' | 'unsaved';  // 可选：状态
}
```

### 3.3 接口继承

使用 `extends` 关键字继承接口：

```typescript
// 基础接口
export interface PathItem {
  repo: string;
  path: string;
}

// 继承 PathItem，添加 name 属性
export interface FileItem extends PathItem { 
  name: string 
}

// FileItem 现在具有 repo、path、name 三个属性
```

### 3.4 接口 vs 类型别名

| 特性 | Interface | Type |
|------|-----------|------|
| 对象类型 | ✅ 推荐 | ✅ 支持 |
| 联合类型 | ❌ 不支持 | ✅ 支持 |
| 继承扩展 | ✅ extends | ✅ 交叉类型 & |
| 声明合并 | ✅ 支持 | ❌ 不支持 |

**最佳实践**：对象形状用 `interface`，其他类型用 `type`。

## 4. 类型别名 (Type Alias)

类型别名使用 `type` 关键字，可以为任何类型起一个新名字。

### 4.1 基本类型别名

```typescript
// 为字符串字面量联合类型起别名
type DocStatus = 'loaded' | 'save-failed' | 'saved' | 'unsaved'

// 为函数类型起别名
type HookFun<T> = (arg: T) => (boolean | void | Promise<boolean | void>)

// 为复杂类型起别名
type PositionState = { line: number, column?: number } | { anchor: string }
```

### 4.2 来自项目的实际例子

**`src/renderer/core/hook.ts`：**

```typescript
// 定义钩子类型
export type HookType = keyof BuildInHookTypes

// 定义钩子函数类型
export type HookFun<T> = (arg: T) => (boolean | void | Promise<boolean | void>)

// 定义钩子结构
export type Hook<T> = {
  fun: HookFun<T>,  // 钩子函数
  once: boolean,     // 是否只执行一次
}
```

## 5. 联合类型与交叉类型

### 5.1 联合类型 (Union Types)

联合类型表示值可以是几种类型之一，使用 `|` 连接：

```typescript
// 状态可以是这四个字符串之一
type DocStatus = 'loaded' | 'save-failed' | 'saved' | 'unsaved'

// 文档类型
type DocType = 'file' | 'dir' | `__${string}`  // 注意模板字面量类型

// 函数返回值可以是多种类型
type HookResult = boolean | void | Promise<boolean | void>
```

**来自项目 `src/renderer/types.ts`：**

```typescript
// 解析链接的结果类型
export type ParseLinkResult = 
  | { type: 'external', href: string }                    // 外部链接
  | { type: 'internal', path: string, name: string, position: PositionState | null }  // 内部链接

// 位置状态可以是三种形式之一
export type PositionState = 
  | { line: number, column?: number }   // 行列位置
  | { anchor: string }                   // 锚点位置
  | PositionScrollState                  // 滚动位置
```

### 5.2 交叉类型 (Intersection Types)

交叉类型将多个类型合并为一个，使用 `&` 连接：

```typescript
// 基础类型
type HasName = { name: string }
type HasAge = { age: number }

// 交叉类型：同时具有 name 和 age
type Person = HasName & HasAge

const person: Person = {
  name: 'Alice',
  age: 25
}
```

## 6. 泛型 (Generics)

泛型是 TypeScript 最强大的特性之一，允许你编写可重用的组件，同时保持类型安全。

### 6.1 基本泛型语法

```typescript
// <T> 是类型参数，可以是任何有效的标识符
function identity<T>(arg: T): T {
  return arg
}

// 使用时指定类型
const result1 = identity<string>('hello')  // result1 是 string
const result2 = identity<number>(42)       // result2 是 number

// TypeScript 也可以自动推断类型
const result3 = identity('world')  // 自动推断为 string
```

### 6.2 泛型接口

**来自项目 `src/share/types.ts`：**

```typescript
// 带泛型参数的接口
export interface LabelValueItem<T> {
  label: string;
  value: T;       // value 的类型由 T 决定
}

// 使用泛型接口
const stringItem: LabelValueItem<string> = {
  label: '名称',
  value: 'Yank Note'
}

const numberItem: LabelValueItem<number> = {
  label: '版本',
  value: 3
}
```

### 6.3 泛型函数

**来自项目 `src/renderer/core/ioc.ts`：**


### 6.4 泛型约束

使用 `extends` 关键字约束泛型参数：

```typescript

  // ...
}

// T 必须是 BuildInIOCTypes 的键
function get<T extends keyof BuildInIOCTypes>(type: T): BuildInIOCTypes[T][] {
  // ...
}

// T 必须有 length 属性
function logLength<T extends { length: number }>(arg: T): number {
  return arg.length
}

logLength([1, 2, 3])    // ✅ 数组有 length

```

### 6.5 keyof 操作符

`keyof` 获取类型的所有键的联合类型：

```typescript
interface Person {
  name: string
  age: number
}

type PersonKeys = keyof Person  // "name" | "age"

// 结合泛型使用
function getProperty<T, K extends keyof T>(obj: T, key: K): T[K] {
  return obj[key]
}

const person = { name: 'Alice', age: 25 }
const name = getProperty(person, 'name')  // 类型是 string
const age = getProperty(person, 'age')    // 类型是 number
```

## 8. async/await 异步编程

### 8.1 Promise 基础

```typescript
// Promise 是一个表示异步操作最终完成或失败的对象
function fetchData(): Promise<string> {
  return new Promise((resolve, reject) => {
    setTimeout(() => {
      resolve('数据加载完成')
    }, 1000)
  })
}

// 使用 .then() 处理结果
fetchData().then(data => console.log(data))
```

### 8.2 async/await 语法

`async/await` 让异步代码看起来像同步代码：

```typescript
// async 函数总是返回 Promise
async function loadDocument(): Promise<string> {
  // await 等待 Promise 完成
  const content = await fetchData()
  return content
}

// 调用 async 函数
async function main() {
  const doc = await loadDocument()
  console.log(doc)
}
```

### 8.3 错误处理

```typescript
async function loadFile(path: string): Promise<string> {
  try {
    const response = await fetch(path)
    if (!response.ok) {
      throw new Error(`HTTP error! status: ${response.status}`)
    }
    return await response.text()
  } catch (error) {
    console.error('加载失败:', error)
    throw error  // 重新抛出错误
  }
}
```

### 8.4 来自项目的实际例子

**`src/renderer/core/hook.ts` 的 triggerHook 函数：**

```typescript
export async function triggerHook<T extends HookType>(
  type: T, 
  arg?: BuildInHookTypes[T], 
  options?: { breakable?: boolean, ignoreError?: boolean }
): Promise<boolean | void> {
  logger.debug('triggerHook', type, arg)
  const items: Hook<any>[] = ioc.get(type)
  
  // 遍历所有钩子函数
  for (const { fun, once } of items) {
    once && removeHook<any>(type, fun)
    
    try {
      if (options?.breakable) {
        // await 等待钩子函数执行完成
        if (await fun(arg)) {
          logger.debug('triggerHook', 'break', fun)
          return true
        }
      } else {
        fun(arg)
      }
    } catch (error) {
      if (options?.ignoreError) {
        console.warn('triggerHook', error)
      } else {
        throw error
      }
    }
  }

  if (options?.breakable) {
    return false
  }
}
```

### 8.5 并行执行多个异步操作

```typescript
// Promise.all：等待所有 Promise 完成
async function loadAllFiles(paths: string[]): Promise<string[]> {
  const promises = paths.map(path => loadFile(path))
  return await Promise.all(promises)
}

// Promise.race：返回第一个完成的 Promise
async function loadWithTimeout(path: string, timeout: number): Promise<string> {
  return Promise.race([
    loadFile(path),
    new Promise<never>((_, reject) => 
      setTimeout(() => reject(new Error('Timeout')), timeout)
    )
  ])
}
```

## 9. 类型工具 (Utility Types)

TypeScript 提供了一系列内置的类型工具。

### 9.1 Partial<T>

将所有属性变为可选：

```typescript
interface Doc {
  name: string
  content: string
  path: string
}

// 所有属性都变为可选
type PartialDoc = Partial<Doc>
// 等价于：{ name?: string; content?: string; path?: string }
```

### 9.2 Required<T>

将所有属性变为必需：

```typescript
interface Config {
  name?: string
  port?: number
}

type RequiredConfig = Required<Config>
// 等价于：{ name: string; port: number }
```

### 9.3 Pick<T, K>

从类型中选取部分属性：

```typescript
interface Doc {
  name: string
  content: string
  path: string
  repo: string
}

type DocPath = Pick<Doc, 'path' | 'repo'>
// 等价于：{ path: string; repo: string }
```

### 9.4 Omit<T, K>

从类型中排除部分属性：

```typescript
type DocWithoutContent = Omit<Doc, 'content'>
// 等价于：{ name: string; path: string; repo: string }
```

## 10. 项目实战分析

### 10.1 IoC 容器类型设计（详解版）

#### 什么是 IoC（控制反转）？

IoC（Inversion of Control，控制反转）是一种设计模式，核心思想是：
- **不直接创建依赖**：组件不直接创建它需要的对象
- **由容器提供依赖**：需要什么，从容器中获取

```
传统方式：    组件A ---(创建)---> 组件B
IoC 方式：    组件A <---(注入)--- IoC容器 ---(管理)---> 组件B
```

#### 完整代码分析

**`src/renderer/core/ioc.ts`：**

```typescript
import type { BuildInIOCTypes } from '@fe/types'

// ==================== 容器存储 ====================
// 使用 Record<string, any[]> 作为底层存储
// key 是类型名称（如 'THEME_STYLES'），value 是该类型的所有注册项数组
const container: Record<string, any[]> = {}

// 版本追踪函数（用于响应式更新）
function updateVersion (items: any) {
  items._version = (items._version || 0) + 1
}

// ==================== 核心 API ====================

// 1. 获取：返回某类型的所有注册项（副本）
export function get<T extends keyof BuildInIOCTypes>(type: T): BuildInIOCTypes[T][] {
  return [...(container[type] || [])]
}

// 2. 获取原始引用（带版本号，用于响应式）
export function getRaw<T extends keyof BuildInIOCTypes>(
  type: T
): (BuildInIOCTypes[T][] & { _version: number }) | undefined {
  return container[type] as (BuildInIOCTypes[T][] & { _version: number })
}

// 3. 注册：向容器添加一个项
export function register<T extends keyof BuildInIOCTypes>(
  type: T, 
  item: BuildInIOCTypes[T]
) {
  if (!container[type]) {
    container[type] = []
  }
  updateVersion(container[type])
  container[type].push(item)
}

// 4. 移除：从容器删除一个项
export function remove<T extends keyof BuildInIOCTypes>(
  type: T, 
  item: BuildInIOCTypes[T]
) {
  if (container[type]) {
    const idx = container[type].indexOf(item)
    if (idx > -1) {
      container[type].splice(idx, 1)
    }
    updateVersion(container[type])
  }
}

// 5. 条件移除：根据条件移除项
export function removeWhen<T extends keyof BuildInIOCTypes>(
  type: T, 
  when: (item: BuildInIOCTypes[T]) => boolean  // 回调函数
) {
  if (container[type]) {
    const items = container[type]
    for (let i = items.length - 1; i >= 0; i--) {
      if (when(items[i])) {
        items.splice(i, 1)
        updateVersion(container[type])
      }
    }
  }
}

// 6. 移除全部
export function removeAll<T extends keyof BuildInIOCTypes>(type: T) {
  if (container[type]) {
    container[type].length = 0
    updateVersion(container[type])
  }
}
```

#### 类型系统深入剖析

##### 第一层：BuildInIOCTypes 类型定义

**来自 `src/renderer/types.ts`：**

```typescript
// BuildInIOCTypes 定义了容器可以存储的所有类型
export type BuildInIOCTypes = 
  // 第一部分：继承所有 Hook 类型（作为键）
  { [key in keyof BuildInHookTypes]: any; } 
  & 
  // 第二部分：额外的 IoC 专用类型
  {
    TABS_ACTION_BTN_TAPPERS: (btns: Components.Tabs.ActionBtn[]) => void;
    TABS_TAB_CONTEXT_MENU_TAPPERS: (items: Components.ContextMenu.Item[], tab: Components.Tabs.Item) => void;
    ACTION_TAPPERS: (action: Action) => void;
    STATUS_BAR_MENU_TAPPERS: any;
    THEME_STYLES: any;
    VIEW_PREVIEWER: Previewer;
    EDITOR_CUSTOM_EDITOR: CustomEditor;
    RENDERERS: Renderer;
    CODE_RUNNER: CodeRunner;
    DOC_CATEGORIES: DocCategory;
    TREE_NODE_ACTION_BTN_TAPPERS: (btns: Components.Tree.NodeActionBtn[], currentNode: Components.Tree.Node) => void;
  }
```

**这个类型做了什么？**

1. **映射类型**：`{ [key in keyof BuildInHookTypes]: any; }`
   - 遍历 `BuildInHookTypes` 的所有键
   - 为每个键创建一个属性，值类型为 `any`

2. **交叉类型**：使用 `&` 合并两个对象类型
   - 结果类型同时拥有两边的所有属性

---

##### 第二层：泛型约束 `T extends keyof BuildInIOCTypes`

```typescript
function get<T extends keyof BuildInIOCTypes>(type: T): BuildInIOCTypes[T][] {
  //     ↑                    ↑
  //     |                    |
  //  类型参数 T           T 必须是 BuildInIOCTypes 的键
}
```

**逐步解析：**

```typescript
// 假设 BuildInIOCTypes 简化为：
type BuildInIOCTypes = {
  THEME_STYLES: any;
  VIEW_PREVIEWER: Previewer;
  CODE_RUNNER: CodeRunner;
}

// keyof BuildInIOCTypes 得到：
type Keys = 'THEME_STYLES' | 'VIEW_PREVIEWER' | 'CODE_RUNNER'

// T extends keyof BuildInIOCTypes 意味着：
// T 只能是 'THEME_STYLES'、'VIEW_PREVIEWER' 或 'CODE_RUNNER'
```

##### 第三层：索引访问类型 `BuildInIOCTypes[T]`

```typescript
function get<T extends keyof BuildInIOCTypes>(type: T): BuildInIOCTypes[T][] {
  //                                                    ↑
  //                                           根据 T 获取对应的值类型
```

**类型推断示例：**

```typescript
// 当调用 get('VIEW_PREVIEWER') 时：
```

#### 设计模式总结

| 概念 | TypeScript 实现 | 作用 |
|------|-----------------|------|
| 类型约束 | `T extends keyof Type` | 限制参数只能是有效的键 |
| 索引访问 | `Type[T]` | 根据键获取对应的值类型 |
| 泛型 | `<T>` | 使函数可以处理多种类型 |
| 交叉类型 | `A & B` | 合并多个类型 |
| 映射类型 | `{ [K in keyof T]: ... }` | 基于已有类型创建新类型 |

这种设计模式被称为 **类型安全的服务定位器（Type-Safe Service Locator）**，是 TypeScript 项目中非常常见的模式。

### 10.2 Hook 系统类型设计（详解版）

#### 什么是 Hook（钩子）系统？

Hook 系统是一种**发布-订阅模式**（也叫观察者模式），允许：
- **注册回调**：在特定事件发生时执行自定义代码
- **解耦模块**：事件触发者不需要知道有谁在监听

```
┌──────────────┐    触发事件     ┌──────────────┐
│   事件源     │ ─────────────→ │  Hook 系统   │
│ (如编辑器)   │                │              │
└──────────────┘                └──────┬───────┘
                                       │ 通知所有订阅者
                    ┌──────────────────┼──────────────────┐
                    ↓                  ↓                  ↓
             ┌──────────┐       ┌──────────┐       ┌──────────┐
             │ 回调函数1 │       │ 回调函数2 │       │ 回调函数3 │
             │ (插件A)  │       │ (插件B)  │       │ (核心模块) │
             └──────────┘       └──────────┘       └──────────┘
```

---

#### 完整代码分析

**`src/renderer/core/hook.ts`：**

```typescript
import type { BuildInHookTypes } from '@fe/types'
import { getLogger } from '@fe/utils/pure'
import * as ioc from './ioc'

// ==================== 类型定义 ====================

// 1. 所有钩子类型的联合
export type HookType = keyof BuildInHookTypes
// 结果：'STARTUP' | 'GLOBAL_RESIZE' | 'GLOBAL_KEYDOWN' | ... 约40种

// 2. 钩子回调函数类型
export type HookFun<T> = (arg: T) => (boolean | void | Promise<boolean | void>)
// 回调可以：返回 boolean（用于中断链）、返回 void、返回 Promise

// 3. 钩子结构
export type Hook<T> = {
  fun: HookFun<T>,  // 回调函数
  once: boolean,     // 是否只执行一次
}

// 4. 无载荷的钩子类型（如 STARTUP、VIEW_RENDER）
export type HookTypeWithoutPayload = { 
  [K in keyof BuildInHookTypes]: BuildInHookTypes[K] extends never ? K : never 
}[keyof BuildInHookTypes]

// 5. 有载荷的钩子类型（如 GLOBAL_KEYDOWN、DOC_SAVED）
export type HookTypeWithPayload = keyof Omit<BuildInHookTypes, HookTypeWithoutPayload>
```

#### 类型系统深入剖析

##### 第一层：BuildInHookTypes 类型定义

**来自 `src/renderer/types.ts`：**

```typescript
export type BuildInHookTypes = {
  // ===== 无载荷的钩子（值为 never）=====
  STARTUP: never,                    // 应用启动
  GLOBAL_RESIZE: never,              // 窗口大小变化
  VIEW_RENDER: never,                // 视图渲染
  VIEW_RENDERED: never,              // 视图渲染完成
  VIEW_MOUNTED: never,               // 视图挂载完成
  
  // ===== 有载荷的钩子（值为具体类型）=====
  GLOBAL_KEYDOWN: KeyboardEvent,     // 全局按键按下
  GLOBAL_KEYUP: KeyboardEvent,       // 全局按键释放
  THEME_CHANGE: { name: ThemeName }, // 主题切换
  DOC_SAVED: { doc: Doc },           // 文档保存后
  DOC_SWITCHED: { doc: Doc | null, opts?: SwitchDocOpts },  // 文档切换后
  
  EDITOR_CONTENT_CHANGE: { uri: string, value: string },    // 编辑器内容变化
  MONACO_READY: { editor: Monaco.editor.IStandaloneCodeEditor, monaco: typeof Monaco },
  
  // ... 约40种钩子类型
}
```

**设计思想：**
- `never` 类型表示"无载荷"，触发时不传递任何数据
- 具体类型（如 `{ doc: Doc }`）表示"有载荷"，触发时传递相关数据

---

##### 第二层：HookFun<T> 泛型函数类型

```typescript
export type HookFun<T> = (arg: T) => (boolean | void | Promise<boolean | void>)
//                  ↑      ↑              ↑
//              类型参数  函数参数      返回值类型
```

**类型实例化示例：**

```typescript
// 当 T = KeyboardEvent 时
type KeydownHandler = HookFun<KeyboardEvent>
// 等价于：(arg: KeyboardEvent) => (boolean | void | Promise<boolean | void>)

// 当 T = { doc: Doc } 时
type DocSavedHandler = HookFun<{ doc: Doc }>
// 等价于：(arg: { doc: Doc }) => (boolean | void | Promise<boolean | void>)

// 当 T = never 时（无载荷）
type StartupHandler = HookFun<never>
// 等价于：(arg: never) => ...  （never 作为参数表示不接受任何值）
```

---

##### 第三层：提取无载荷钩子类型（核心难点）

```typescript
export type HookTypeWithoutPayload = { 
  [K in keyof BuildInHookTypes]: BuildInHookTypes[K] extends never ? K : never 
}[keyof BuildInHookTypes]
```

**这是最复杂的类型操作，让我们逐步拆解：**

**步骤 1：映射类型遍历所有键**

```typescript
{ [K in keyof BuildInHookTypes]: ... }

// K 依次取值：'STARTUP', 'GLOBAL_RESIZE', 'GLOBAL_KEYDOWN', ...
```

**步骤 2：条件类型判断**

```typescript
BuildInHookTypes[K] extends never ? K : never

// 对于每个 K，检查其值类型：
// - 如果值类型是 never（无载荷），返回 K（键名）
// - 如果值类型不是 never（有载荷），返回 never
```

**步骤 3：构建中间类型**

```typescript
// 假设简化的 BuildInHookTypes：
type BuildInHookTypes = {
  STARTUP: never,           // 无载荷
  GLOBAL_KEYDOWN: KeyboardEvent,  // 有载荷
  VIEW_RENDER: never,       // 无载荷
  DOC_SAVED: { doc: Doc },  // 有载荷
}

// 映射后得到：
type IntermediateType = {
  STARTUP: 'STARTUP',       // never → 返回键名
  GLOBAL_KEYDOWN: never,    // KeyboardEvent → 返回 never
  VIEW_RENDER: 'VIEW_RENDER', // never → 返回键名
  DOC_SAVED: never,         // { doc: Doc } → 返回 never
}
```

**步骤 4：索引访问提取值的联合**

```typescript
IntermediateType[keyof BuildInHookTypes]
// = IntermediateType['STARTUP' | 'GLOBAL_KEYDOWN' | 'VIEW_RENDER' | 'DOC_SAVED']
// = 'STARTUP' | never | 'VIEW_RENDER' | never
// = 'STARTUP' | 'VIEW_RENDER'  （never 在联合类型中被消除）
```

**最终结果：**

```typescript
type HookTypeWithoutPayload = 'STARTUP' | 'GLOBAL_RESIZE' | 'VIEW_RENDER' | 'VIEW_RENDERED' | ...
// 只包含无载荷的钩子类型
```

---

##### 第四层：提取有载荷钩子类型

```typescript
export type HookTypeWithPayload = keyof Omit<BuildInHookTypes, HookTypeWithoutPayload>
```

**拆解：**

```typescript
// Omit<T, K> 从 T 中移除键 K
Omit<BuildInHookTypes, HookTypeWithoutPayload>
// = 移除所有无载荷的键后剩余的类型
// = { GLOBAL_KEYDOWN: KeyboardEvent, DOC_SAVED: { doc: Doc }, ... }

// keyof 获取所有键
keyof Omit<...>
// = 'GLOBAL_KEYDOWN' | 'DOC_SAVED' | ...
```

##### 第五层：函数重载实现类型安全

```typescript
// 重载签名 1：无载荷钩子，不传参数
export async function triggerHook<T extends HookTypeWithoutPayload>(
  type: T
): Promise<void>

// 重载签名 2：无载荷钩子，可中断
export async function triggerHook<T extends HookTypeWithoutPayload>(
  type: T, 
  arg: undefined, 
  options: { breakable: true }
): Promise<void>

// 重载签名 3：有载荷钩子，必须传参数

  type: T, 
  arg: BuildInHookTypes[T]
): Promise<void>

// 重载签名 4：有载荷钩子，可中断，返回 boolean
export async function triggerHook<T extends HookTypeWithPayload>(
  type: T, 
  arg: BuildInHookTypes[T], 
  options: { breakable: true, ignoreError?: boolean }
): Promise<boolean>

// 重载签名 5：有载荷钩子，不可中断
export async function triggerHook<T extends HookTypeWithPayload>(
  type: T, 
  arg: BuildInHookTypes[T], 
  options?: { breakable?: false, ignoreError?: boolean }
): Promise<void>

// 实现签名（内部使用，对外不可见）
export async function triggerHook<T extends HookType>(
  type: T, 
  arg?: BuildInHookTypes[T], 
  options?: { breakable?: boolean, ignoreError?: boolean }
): Promise<boolean | void> {
  // 实现代码...
}
```

**函数重载的作用：**

```typescript
// ✅ 正确：无载荷钩子不传参数
triggerHook('STARTUP')
triggerHook('VIEW_RENDER')

// ✅ 正确：有载荷钩子必须传对应类型的参数
triggerHook('GLOBAL_KEYDOWN', keyboardEvent)
triggerHook('DOC_SAVED', { doc: myDoc })

// ❌ 错误：无载荷钩子不能传参数
triggerHook('STARTUP', someData)  // 编译错误

// ❌ 错误：有载荷钩子参数类型不匹配
triggerHook('DOC_SAVED', keyboardEvent)  // 编译错误，应该传 { doc: Doc }

// ✅ 可中断的钩子返回 boolean
const interrupted = await triggerHook('GLOBAL_KEYDOWN', event, { breakable: true })
if (interrupted) {
  console.log('有处理器中断了事件链')
}
```

#### 设计模式总结

| 概念 | TypeScript 实现 | 作用 |
|------|-----------------|------|
| 映射类型 | `{ [K in keyof T]: ... }` | 遍历类型的所有键 |
| 条件类型 | `T extends U ? X : Y` | 根据条件选择类型 |
| never 消除 | `'A' \| never = 'A'` | 在联合类型中过滤 |
| 索引访问 | `T[K]` | 获取类型的属性类型 |
| 函数重载 | 多个函数签名 | 根据参数提供不同返回类型 |
| Omit | `Omit<T, K>` | 从类型中移除指定键 |

这种设计模式实现了**完全类型安全的事件系统**，是 TypeScript 高级类型的经典应用！
